## Architecture and Design

### Technology Stack
- **Language**: Python 3.13+
- **Dependencies**: yt-dlp, gallery-dl, telethon, requests, python-telegram-bot
- **Runtime**: Docker (docker-compose with bot + nginx services)
- **CI/CD**: GitHub Actions (test -> deploy pipeline on push to master)
- **Infrastructure**: Oracle Cloud (158.180.23.247), SSH deploy
- **Video Processing**: ffmpeg/ffprobe for H.264 transcoding

### Key Architectural Decisions
- **Dual entry point**: CLI (`download.py`) and Telegram bot (`bot.py`) share the same download engine
- **Platform dispatch**: `detect_platform(url)` returns platform string, each platform has dedicated URL parser and download function
- **Download backends**: yt-dlp for Twitter video, gallery-dl for images/Instagram, telethon for Telegram
- **Large file handling**: Files >50MB served via nginx on port 9090 instead of Telegram upload
- **Channel downloads**: Async with 10-concurrent semaphore, run via `run_in_executor()` from bot context
- **Access control**: Username allowlist + phone verification via `/auth` command
